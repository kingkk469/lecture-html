# lecture-html · 讲课 HTML 设计公式

把「一份讲稿要点」变成「一份能演示、能分享、能发布的 HTML 幻灯片」。

## 何时使用这个 skill

触发场景(用户的原话可能是):

- "帮我做一份讲课 PPT"
- "把这份讲稿做成幻灯片"
- "生成一份分享用的演示页"
- "我要讲课 / 直播 / 开发布会,需要一份 slide"
- "做一个知识付费课程的开场封面页"
- "做一份能发到 GitHub Pages 的演示页"
- 用户提到"乘势"/"中式黑金"/"古典+科技"风格

## 适合什么样的内容

- AI / 商业 / 创业 / 知识付费类课程
- 技术分享 / 公司战略 / 行业洞察
- 开课导言 / 章节封面 / 数据展示
- 中文为主(标题也支持中英混排)

## 视觉风格(默认)

- **基调**:近黑底 (`#020208`) + 古金 (`#c8a35c`) + 红色强调 (`#d84a3a`) + 米白文字 (`#ebe5d5`)
- **气质**:中式玄学 × 科技未来 — 像帝王玉玺 + 数字星图
- **装饰**:左侧云纹+电路 / 右上金弧 / 左下水波+红印章 / 右下星图
- **字体**:Noto Serif SC 思源宋体 (主) + Cinzel (数字)

## 资源文件

| 文件 | 用途 |

| `template.html` | 空白可填模板 — 复制改字即可 |

## 怎么用(给 Claude 的步骤)

### 步骤 1:了解用户的讲课内容

问清楚:
- 主题 / 章节 / 听众是谁
- 每页要讲什么核心论点
- 是否有具体数据 / 案例 / 对比 / 时间节点

### 步骤 2:为每个论点选 slide 类型

8 种核心类型:

| 类型 | 何时用 | CSS class |
|:---|:---|:---|
| **封面 cover** | 开场 / 结尾 | `.slide-cover` |
| **章节标题 title** | 分章节 | `h1.title` + `h2.subtitle` |
| **大数字 bignum** | 让数据说话 | `.bignum` |
| **时间轴 timeline** | 展示变化前后 | `.timeline` |
| **对比 vs** | A vs B 数据 | `.vs` |
| **列表 list** | 3-5 个并列要点 | `.list-big` |
| **卡片 card** | 2-3 栏选项(三条路 / 三方向) | `.three-col + .col-card` |
| **金句 quote** | 一句话戳到点 | `.quote-big` 或 `.quote-mid` |

### 步骤 3:复制 template.html,挨个填 slide

`template.html` 里已经有 8 种类型的占位 slide,改字即可。

### 步骤 4:如果需要自定义背景/logo

调用 `scripts/generate_images.py`:
- 设置 `OPENAI_API_KEY` 环境变量
- 在 TASKS 列表里改 prompt
- 跑脚本生成新图,替换 `images/` 里的对应文件

### 步骤 5:浏览器打开 → F 全屏 → 开讲

## 设计规则(强制)

1. **一页一个核心** — 不堆信息,信息密度比传统 PPT 低 30%
2. **大字号** — 标题 ≥ `clamp(2.4rem, 5.5vw, 5rem)`,后排能看清
3. **克制装饰** — 所有装饰只在角落,中心大段留黑
4. **不用 emoji** — 渲染不一致,用几何字符(★ ① ② ③ ◆)替代
5. **品牌条** — 全局底部居中 logo + 课程名,所有 slide 共享

## 反例(常犯的错)

- ✗ 用 PowerPoint 风格的"一页 10 条要点 + 5 张配图"
- ✗ 用 emoji 做图标(渲染跨平台不一致)
- ✗ 标题用 16px(投影看不清)
- ✗ 一页塞 3 个论点(应该拆 3 页)

## 输出文件位置建议

- 单个讲课页:`<topic>.html`(放工作目录)
- 多章节课:`<course>/<chapter>.html`(分文件夹)
- 共享背景/logo:`images/`(所有 HTML 共用)
